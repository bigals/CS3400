HTML Local Storage

HTML Local storage can allow HTML to store data locally within the user's browser, unlike cookies the storage limit is at least 5MB, and data is never transferred to the server. The data is stored as name/value pairs, and the storing web page is the only one that can acsess the data. window.localStorage keeps data with no expiration date, where as code.sessionStorage stores data only until the tab is closed.

HTML App Cache

The "< html manifest="demo.appcache">" element, placed in the < head > element of the page, will allow the webpage to be cached, and then brought up again, even when offline. Then the manifest file should have the following form, the CACHE MANIFEST files are those that are available offline and is mandatory in the file, the NETWORK files are those that should not be available offline, and the FALLBACK is the file that should be loaded to replace the entire "/html/" folder with the offline.html file. NETWORK, and FALLBACK are optional fields.

Example CacheManifest file
CACHE MANIFEST
/index.html
/styles.css
/scripts.js
NETWORK:
login.asp
FALLBACK:
/html/ /offline.html

HTML SSE

A server-sent event is when a web page automatically gets updates from a server. This was possible before HTML5, but the web page would have to ask if any updates were available. With server-sent events, the updates come automatically. Examples: Facebook/Twitter updates, stock price updates, news feeds, sport results, etc.

Accessibility means helping people with disabilites utilize all of the functionality of user interfaces. The W3C Web Accessibility Initiative (WAI) lays out international standards of inclusive design for a broad range of web users and people who use the web.

Assistive Technologies

Assistive Technologies are devices modified to improve functional capabilities of people with disabilities. Software must be compatible with the current assistive technologies.

Cognitive impairments

Persons with learning disabilities often have trouble navigating complicated interfaces.

  • Input assistance can help useres avoid and correct mistakes.
  • Users should be given extra time on applications with time limits. This can be done by giving the user the option to adjust time limits
  • Navigation should be kept as consistent and simple as possible with few links and visually distinct options.
  • The interface should appear and operate in predictable ways
  • Content

Visual impairment

  • Alternitive text (alt attribute in html) should be used to accomidate screen readers. Every image should have alt text which describes the function and content
  • Text can be converted to extra large print
  • Convert text to braille

Hearing-related disabilities

  • Sign language videos can be used in place of audio files
  • Caption and/or provide scripts for media
  • Speech-to-text technology can interpret audio for the user

Motor or dexterity impairment

  • Make JavaScript handlers device independent, so they can accept inputs other than a mouse
  • Common alternative input devices include switches, joysticks, and trackballs
  • Provide all functionality from all types of input devices/li>

Elderly Users

As people age their senses decline. They will likely have problems with vision, physical ability (dexterity and fine motor control), hearing, and cognitive ability. Audio

Audio: Elderly people often have trouble hearing low volumes and high pitches. This should be taken into consideration when choosing audio for your interface. Vision: Older users will usually have a tougher time identifying contrast and their perception of colors will change. Distinct colors can be used to make it easier for people to distinguish between elements on the page. Motor Skills: Elderly users can have trouble using a mouse or keyboard. They often will have difficulty clicking small areas. It is important to consider size and motor skills if older people will be using the interface. Cognitive Decline: Navigation, comprehension, and task completion can be difficult for older users with declining cognitive ability. These users may have short term memory problems, difficulty concentrating, and may become distracted easily. The developer should be sure to keep the interface simple and consistent to help people with cognitive deficiencies.

Accessibility APIs

Software APIs exist to allow assistive technology products to work with software.
  • Java Accessibility and the Java Access Bridge
  • Assistive Technology Service Provider Interface on UNIX and Linux
  • IAccessible2 on Microsoft Windows
  • Mac OS X Accessibility
  • Microsoft UI Automation
